## 🤔 为什么调整？

### 更符合用户心智模型

**模型评测 = AI工作负载**
- 需要运行计算任务
- 消耗GPU/CPU资源
- 有执行状态（等待/运行/完成）

**Pipeline = AI工作流编排**
- 编排多个AI任务
- 串联训练→评测→部署
- 管理端到端流程

### 更清晰的信息架构

**之前的问题**:
- 训练任务在"AI工作负载"
- 评测任务在"数据资产"
- 用户需要跨分组操作 ❌

**现在的优势**:
- 所有AI任务都在"AI工作负载"
- 操作流程更连贯 ✅

### 对标行业标准

**AWS SageMaker**:
```
├── Training Jobs
├── Model Evaluation ← 在工作流中
└── Pipelines
```

**Google Vertex AI**:
```
├── Training
├── Evaluation ← 在工作流中
└── Pipelines
```

## ✨ 用户体验提升

### 场景：完整的模型开发流程

**之前**:
```
1. 数据集 (数据资产)
2. 训练任务 (AI工作负载) ← 切换分组
3. 模型评测 (数据资产) ← 再次切换
4. 推理部署 (AI工作负载) ← 又要切换
```

**现在**:
```
1. 数据集 (数据资产)
2. 训练任务 (AI工作负载)
3. 模型评测 (AI工作负载) ← 同一分组
4. 推理部署 (AI工作负载) ← 同一分组
```

或直接使用:
```
Pipeline编排 (AI工作负载)
  ├─ 数据预处理
  ├─ 模型训练
  ├─ 模型评测
  └─ 模型部署
```
